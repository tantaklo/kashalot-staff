# Карта документации — ops-site

Правило: **один факт — одно каноническое место**.

## Где что лежит

| Что | Файл | Назначение |
|---|---|---|
| Правила работы с проектом | [`../CLAUDE.md`](../CLAUDE.md) | Стек, деплой, что не ломать |
| Долговечная память | [`project-memory.md`](project-memory.md) | Факты, аудитория, контент, словарь |
| Баги и gotchas | [`bugs-and-gotchas.md`](bugs-and-gotchas.md) | Что ломалось и как чинить |
| Журнал решений | [`decision-log.md`](decision-log.md) | Принятые решения — не спорить заново |

## Структура проекта

```
ops-site/
├── CLAUDE.md                  — правила работы
├── docs/                      — документация (этот раздел)
├── src/
│   └── content/docs/          — весь контент сайта (.md файлы)
│       ├── start/             — С чего начать
│       ├── daily/             — Подготовка смены, замена АКБ, проверки
│       ├── battery/           — АКБ: правила, хранение, экстренные случаи
│       ├── troubleshooting/   — Снегокат не едет, износ салазок
│       ├── weather/           — Погода и покрытие
│       ├── weekly/            — Еженедельные проверки
│       ├── videos/            — Видеоинструкции (плейсхолдеры)
│       └── support/           — Контакты и поддержка
├── astro.config.ts            — настройки сборки
└── public/                    — статические файлы (логотип, favicon)
```

## Как обновлять

| Событие | Куда писать |
|---|---|
| Стабильный факт об аудитории или контенте | `project-memory.md` |
| Что-то сломалось при деплое или редактировании | `bugs-and-gotchas.md` |
| Принято решение по структуре, дизайну, домену | `decision-log.md` |
| Изменился стек или правила работы | `CLAUDE.md` |

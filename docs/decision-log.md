# Decision log — ops-site

Принятые решения, которые не нужно пересматривать без причины.

---

## Решения

### 2026-05 — GitHub Pages вместо Netlify

- **Контекст:** Сайт задеплоен на оба хостинга, Netlify оказался недоступен в РФ
- **Решение:** Основной URL — GitHub Pages. Netlify не использовать и не рекламировать.
- **Почему так:** Аудитория — сотрудники в России, часто на телефоне в поле
- **Альтернативы:** VK Cloud, Selectel — избыточны для статики
- **Когда пересмотреть:** Если появится кастомный домен staff.k-s.app

### 2026-05 — Astro + astro-theme-pure

- **Контекст:** Нужен статический сайт-справочник, быстрый на мобильных
- **Решение:** Astro 6, тема astro-theme-pure (cworld1), output: 'static'
- **Почему так:** Тема оптимизирована для документации, быстрая загрузка,
  хорошая навигация и читаемость на мобильных, не нужен backend
- **Альтернативы:** VitePress, Docusaurus — тяжелее; Hugo — менее гибкий
- **Когда пересмотреть:** Если понадобится авторизация или динамический контент

### 2026-05 — Только операционка в v1, без продаж и истории

- **Контекст:** Соблазн добавить историю компании и инвест-цифры
- **Решение:** v1 — только операционный контент для сотрудника. Ничего лишнего.
- **Почему так:** Сотрудник открывает сайт чтобы решить задачу, не читать о компании
- **Альтернативы:** Полный сайт сразу — замедляет запуск и размывает фокус
- **Когда пересмотреть:** После того как v1 работает и сотрудники им пользуются

### 2026-05 — Самодостаточные разделы, без вложенной навигации

- **Контекст:** Аудитория открывает сайт на морозе в перчатках
- **Решение:** Каждый раздел самодостаточен, открывается по прямой ссылке.
  Навигация плоская, без вложенных меню.
- **Почему так:** Найти нужное за 10 секунд — главный UX-принцип
- **Альтернативы:** Иерархическая навигация — слишком много тапов в перчатках
- **Когда пересмотреть:** Если контент вырастет настолько, что плоская навигация перестанет работать

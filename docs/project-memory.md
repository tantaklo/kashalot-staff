# Project memory — ops-site

Долговечные факты о проекте. Только то, что полезно через неделю или месяц.

---

## Проект

- **Название:** База знаний сотрудника Кашалот (ops-site)
- **Что строим:** Статический сайт-справочник для сотрудников франчайзи
- **Для кого:** 1–3 сотрудника на франчайзи — «руки бизнеса»
- **Главная цель:** Новый сотрудник читает и думает: «Всё понятно, можно работать»
- **Что важно не сломать:** `astro.config.ts`, `src/content/docs/`, GitHub Actions

---

## Аудитория

Два режима использования сайта:
- **Дома перед сменой** — читает целиком, получает общую картину
- **В поле** — ищет конкретную инструкцию прямо на улице, в перчатках, на морозе

Требования к подаче: самодостаточные разделы, крупные заголовки, нумерованные шаги, найти нужное за 10 секунд.

**Ядро операционки — управление батареями АКБ.** Всё остальное вокруг этого.

## Долговечные факты

### 2026-05 — Netlify заблокирован в России

- **Факт:** Netlify недоступен из России без VPN
- **Почему важно:** Целевая аудитория — сотрудники в России на телефоне
- **Решение:** Только GitHub Pages (tantaklo.github.io/kashalot-staff)

### 2026-05 — Исходная база знаний

- **Факт:** Оригинальный документ «База знаний Кашалот.docx» перенесён в
  `Knowledge/kashalot-kb.md` (326 строк). Контент сайта написан на его основе.
- **Почему важно:** Если нужно добавить новый раздел — сначала смотреть в KB
- **Разделы KB для сотрудника:** 2, 3, 4, 5, 6, 8. Многие помечены 🔲 (неполные)

### 2026-05 — Деплой занимает ~1 минуту

- **Факт:** После `git push` в main GitHub Actions собирает и деплоит сайт.
  Среднее время — около 1 минуты.
- **Почему важно:** Не пугаться, если сайт не обновился сразу после push
